# Diary-Of-Gamers

Мой проект «Дневник Геймера» представляет собой реляционную базу данных, спроектирована под онлайн сервис типо "Shikimori", для того, чтобы предоставлять возможность хранить данные о множестве пользователей и, например собирать статистику. Проект включает в себя: Нормализованную схему данных в третьей нормальной форме (3НФ), охватывающую сущности: пользователи, игры, жанры, жанр каждой игры, разработчики, разработчики каждой конкретной игры, дневник пользователя, любимые игры, игровые обзоры.

Таблицы:

Genres - Хранит справочник игровых жанров. Поля: genre_id - уникальный идентификатор жанра (первичный ключ), name - название жанра (например, «Action», «RPG», «Strategy»). Обеспечивает нормализацию: избегаем дублирования названий жанров у игр.

Games - Каталог видеоигр с основной информацией. Поля: game_id - уникальный ID игры (первичный ключ), title - название игры, description - описание, release_year - год выхода, age_rating - возрастной рейтинг, metacritic_rating - оценка на Metacritic, created_at - дата добавления в базу. Центральная сущность, от которой зависят многие связи.

Users - Информация о зарегистрированных пользователях платформы. Поля: user_id - уникальный ID пользователя (первичный ключ), email - электронная почта, password_hash - хэш пароля, nickname - игровой псевдоним, created_at - дата регистрации. Уникальные ограничения на email и nickname предотвращают дублирование.

Developers - Справочник игровых студий и отдельных разработчиков. Поля: developer_id - уникальный ID разработчика (первичный ключ), name - имя разработчика или название студии. Вынесено в отдельную таблицу для соблюдения 3НФ: данные о разработчике не дублируются в каждой игре.

Games_Genres - Промежуточная таблица для связи многие-ко-многим между играми и жанрами. Поля: game_id - ссылка на игру, genre_id - ссылка на жанр. Составной первичный ключ (game_id, genre_id). Позволяет одной игре принадлежать к нескольким жанрам.

Game_Developers - Промежуточная таблица для связи многие-ко-многим между играми и разработчиками. Поля: game_id - ссылка на игру, developer_id - ссылка на разработчика, role_name - роль разработчика в проекте. Составной первичный ключ (game_id, developer_id). Учитывает, что над одной игрой может работать несколько разработчиков.

Diary - Фиксирует, в какие игры и когда играли пользователи. Поля: user_id - ссылка на пользователя, game_id - ссылка на игру, playing_date - дата игровой сессии. Составной первичный ключ (user_id, game_id, playing_date). Центральная таблица для анализа игровых привычек.

Favourite_Games - Хранит избранные игры пользователей. Поля: user_id - ссылка на пользователя, game_id - ссылка на игру, date_to_add - дата добавления в избранное. Составной первичный ключ (user_id, game_id). Отражает персональные предпочтения игроков.

Game_Review - Содержит оценки и текстовые отзывы пользователей об играх. Поля: review_id - уникальный ID отзыва (первичный ключ), user_id - ссылка на пользователя, game_id - ссылка на игру, rating - оценка от 1 до 10, review - текст отзыва, created_at - дата создания. Уникальное ограничение (user_id, game_id) гарантирует один отзыв на игру от пользователя.

Game_Review - Содержит оценки и текстовые отзывы пользователей об играх.
Поля: review_id — уникальный ID отзыва (первичный ключ), user_id — ссылка на пользователя, game_id — ссылка на игру, rating — оценка от 1 до 10, review — текст отзыва, created_at — дата создания. Уникальное ограничение (user_id, game_id) гарантирует один отзыв на игру от пользователя.

Все таблицы связаны внешними ключами, не содержат избыточных данных и соответствуют третьей нормальной форме (3НФ).  Проект позволяет не только хранить данные, но и выполнять аналитические запросы для анализа игровых предпочтений. Три аналитических запроса: Игры с наибольшим количеством записей в дневнике, Пользователи с массивом их любимых игр в формате JSON, ТОП-5 самых популярных игр за месяц. Визуальную ER-диаграмму, созданную в draw.io, которая отображает структуру таблиц и связи между ними.
